To the right is a javascript plugin ( designdoc ) that reads a list of events from the embedding application, and then performs some visualization. It demonstrates that a plugin can

  • read information via an application provided api (see plugin-api.js)
  • generate a user interface using html templating, and CSS
  • perform calculations and manipulate styles to update that application
  • schedule periodic tasks
  • respond to user actions (click on a node) and send state back to the application
